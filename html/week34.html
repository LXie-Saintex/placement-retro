On Tuesday, we had a fascinating discussion around the testing process & Jira workflow in general. Because the majority of the tasks on the board rely on MAIN to implement their part, we had the time to reflect on our current ways of working. We had 2 sessions which really unveiled a lot of valid doubts and confusions about the CI. I had a better understanding of the status quo and found out what has been confusing me since I joined the team.

  1. manual tests were done across the environments and even on PR stage. The complexity of the work is significantly increased as most of the repos do not have up to date documentations on how to run the projects and almost exclusively rely on low-level plumbing.
  2. the development process is not test-driven. When the code goes into the CI for the first time, there is no test for it.
  3. this is further complicated by the abusive usage of git branching for division of labour. Instead of being an atomic piece, multiple PRs rely on each other for final completion where the program correctness is confirmed.
    4. >
  4. as a result, the code review process is prolonged by the need for piecing together different PRs which often carries little description and requires the reviewer going back and forth between Jira and Github.
  5. Code review lasts even longer when the commit history is not well-groomed to reflect the "true" changes.

The team tried to introduce more environment which is a non-solution to the real underlying problem. Besides, someone told me that creating a "true" PR environment is not worth it and its benefits questionable compared to simple mocked integration tests. I expressed my thoughts on this and argued that it's mainly because we are understaffed to set up and maintain PR environments, which can be a constant source of pain from my experience at Sun Savers. I added the team to the CI channel and also started to config a new CI using the existing capabilities shared within the company. I added to README the links to those central repos and hope they will be used more often. I have the impression that the repos are not visible to teams outside MAIN and this seems bad.
