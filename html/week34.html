On Tuesday, we had a fascinating discussion around the testing process & Jira workflow in general. Because the majority of the tasks on the board rely on MAIN to implement their part, we had the time to reflect on our current ways of working. We had 2 sessions which really unveiled a lot of valid doubts and confusions about the CI. I had developed a better understanding of the status quo and found out what has been confusing me since I joined the team.

  1. manual tests were done across the environments and even on PR stage. The complexity of the work is significantly increased as most of the repos do not have up to date documentations on how to run the projects and almost exclusively rely on low-level plumbing.
  2. the development process is not test-driven. When the code goes into the CI for the first time, there is no test for it which sort of defeats the purpose of having a CI.
  3. the process is further complicated by the abusive usage of git branching for coordinating workload. Instead of being an atomic piece, the PRs rely on each other for the confirmation of program correctness.
  4. as a result, the code review process requires the reviewer piecing together different PRs and going back and forth between Jira and Github just to figure out what does the PR achieve.
  5. Code review is further prolonged when the commit history is not well-groomed to reflect the "true" changes since the last stable version.

The team tried to introduce more environment which is a non-solution to the actual problem. Besides, someone told me that it's questionable whether it brings more benefits than simple mocked integration tests. I shared my opinions with the team and argued that we are understaffed to set up and maintain PR environments, which can be a constant source of distraction based on my previous experience. I added the team to the CI discussion channel and also started to config a new CI using the existing capabilities shared within the company. I added to README the links to those central repos and hope they will be used more often. I have the impression that the repos are not visible to teams outside MAIN and this seems bad.
